# dynamic-ips — Online Dial-a-Ride with Column Generation

Real-time dispatching for large-scale ride-sharing using a rolling-horizon
column generation (CG) solver.  At each epoch the master problem is solved
as a set-partitioning LP/MIP; routes are generated by a labelling /
ESPPRC sub-problem.  Two algorithmic workflows are implemented:

| Workflow | Description |
|----------|-------------|
| **B-CG** (Batch CG) | One optimal solve per epoch |
| **A-CG** (Anytime CG) | Iterative improvement within each epoch with quality guarantees |

## Datasets

Both benchmark sets are derived from NYC Taxi and Limousine Commission (NYCTLC)
trip records.

| Dataset | Network | Zenodo |
|---------|---------|--------|
| **NYC-DARP** (this work) | Custom Manhattan network (OSM-derived virtual stops) | *(link to be added upon publication)* |
| **Riley et al. (2019)** | 1 718-cell Manhattan grid | [doi:10.5281/zenodo.18745880](https://doi.org/10.5281/zenodo.18745880) |

Raw trip records and network files (stop locations, travel-time matrices)
are stored in Zenodo.  This repository contains only the code; generated
benchmark instances are included in `DARP_IPS/`.
